Powershell language uniquement pour scrippting, donc pas un shell
Créé par Microsoft en 2006 puis open source en 2016 (pour remplacer cmd.exe) basé sur le framework .NET
Compatibilité POSIX, script uniquement et orienté objet. 
Chaque commande retourne un objet, les objets ont des attributs associés (évite parsing). $_ représente l'attribut créé
Variable d'environnement ne sont pas des variables Powershell

## Cmdlets
Ne sont pas liés à un logiciel. Des abstractions vers des systèmes existants.  
  
Voir la doc de group-object pour l'exo   
  
Découvrir des packages de scripts powershell : https://www.powershellgallery.com/packages/Wsus-Maintenance/
